# PaPaPhone

Голосовой телефон для людей с потерей зрения. Полностью офлайн, управление только голосом.

Работает на Orange Pi Zero 2W + SIM7600G-H 4G HAT. Можно адаптировать под любой Linux.

## Как это работает

Дедушка говорит **"телефон позвони маме"** -- одной фразой. Система распознаёт речь, находит контакт по имени (с учётом склонений: "маме" = "мама"), переспрашивает подтверждение и звонит. Всё офлайн, без интернета (кроме VoIP-звонков).

```
Звонки → VoIP (SIP через Zadarma/Sipnet) или AT-модем
SMS    → SIM7600 (AT-команды)
Голос  → Vosk (офлайн) + Piper TTS (офлайн)
```

Архитектура -- конечный автомат (FSM) с пятью состояниями. Входящие звонки и SMS обнаруживаются автоматически фоновыми потоками.

## Голосовые команды

После wake-фразы **"телефон"** (или в одной фразе с ней):

### Звонки

| Команда | Пример | Что происходит |
|---------|--------|----------------|
| Позвонить | "позвони маме", "позвонить сыну" | Поиск по имени/прозвищу (морфология) -> подтверждение -> звонок |
| Набрать номер | "набери номер 89001234567" | Подтверждение -> набор |
| Ответить | "ответь", "ответить" | Принять входящий |
| Положить трубку | "положи трубку", "положить трубку" | Завершить звонок |

### SMS

| Команда | Пример | Что происходит |
|---------|--------|----------------|
| Сообщения | "сообщения" | Все SMS по одному: дальше / позвонить / хватит |
| Новые сообщения | "новые сообщения" | Только непрочитанные |
| *(автоматически)* | | Днём SMS озвучиваются сразу. Ночью (23:00--9:00) копятся, утром: "У вас N непрочитанных". Часы настраиваются в `.env` |

### Контакты

| Команда | Пример | Что происходит |
|---------|--------|----------------|
| Найти контакт | "найди контакт маму" | Поиск -> позвонить / прозвище / переименовать |
| Добавить контакт | "добавь контакт" | Имя -> номер по цифрам -> прозвище (необязательно) |
| Прозвище | "прозвище маме" | Добавить альтернативное имя к контакту |
| Список контактов | "список контактов" | Зачитать все контакты |
| Экстренные контакты | "экстренные контакты" | Кому уйдёт SMS при вызове 112 |

### Информация

| Команда | Пример | Что происходит |
|---------|--------|----------------|
| Время | "который час", "сколько времени" | Текущее время |
| Адрес | "адрес" | Озвучить домашний адрес |
| Сигнал | "сигнал", "уровень сигнала" | Качество связи |
| Последние звонки | "последние звонки" | Последние 5 вызовов с именем/номером и временем |
| Громче | "громче" | +10%, озвучивает уровень. Максимум 100% |
| Тише | "тише" | -10%, озвучивает уровень. Минимум 10% |
| Команды | "команды" | Список всех функций |

### Экстренные

| Команда | Пример | Что происходит |
|---------|--------|----------------|
| Экстренный вызов | "спасите", "скорая", "мне плохо", "помогите", "нужна помощь" | Звонок на SOS-номер (112). Тишина = звонить. SMS экстренным контактам с адресом |

При экстренном вызове:
1. SMS с адресом отправляется всем **экстренным контактам** (is_emergency) -- до набора
2. Набирается **SOS-номер** (контакт "Скорая", is_sos=1) -- создаётся в БД автоматически

### Управление

| Команда | Пример | Что происходит |
|---------|--------|----------------|
| Отмена | "стоп", "отмена" | Прервать текущее действие (с подтверждением) |

### Секретное меню

Вход по фразе **"секретное меню"**, далее называешь имя контакта:

- **"удалить"** -- удалить контакт
- **"экстренный"** -- пометить как экстренный (получит SMS при вызове 112)
- **"убрать из экстренных"** -- снять пометку

Другие служебные команды:
- **"очистить звонки"** -- очистить историю вызовов

## Входящие звонки

Система опрашивает провайдер звонков каждые 2 секунды. Определяет кто звонит по контактам. Спрашивает "Входящий от Мама. Ответить?" Молчание = сброс (безопасный дефолт).

## Провайдер звонков

Звонки идут через абстракцию `CallProvider` -- сценарии не знают, VoIP это или AT-модем.

| Режим | Настройка | Как работает |
|-------|-----------|--------------|
| **VoIP** (по умолчанию) | `CALL_MODE=voip` | SIP через Zadarma/Sipnet. Аудио через ALSA Orange Pi |
| **AT-модем** | `CALL_MODE=modem` | AT-команды через SIM7600. Fallback если нет VoIP |
| **Демо** | `--demo` | Звонки симулируются. Для отладки |

SIM7600 остаётся для SMS и 4G-интернета в любом режиме.

## Нечёткий поиск

Vosk часто коверкает слова. Если точный матч не нашёл команду, включается fuzzy:

1. **Левенштейн** (rapidfuzz) -- сравнение по буквам, ловит опечатки ASR
2. **Эмбеддинги** (navec) -- семантическая близость, fallback

Работает и для команд, и для имён контактов ("игарю" -> "Игорь").

## Конечный автомат (FSM)

```
IDLE -> LISTENING -> EXECUTING -> IDLE
  |        |            |
  |        |            +-> IN_CALL -> IDLE
  |        |
  +-> INCOMING -> IDLE
```

- **IDLE** -- ждёт wake-фразу, проверяет входящие звонки и SMS
- **LISTENING** -- wake получен, ждёт команду (12 с)
- **EXECUTING** -- сценарий выполняется
- **IN_CALL** -- активный звонок, слушает "положи трубку"
- **INCOMING** -- обработка входящего звонка

## Технологии

| Компонент | Технология |
|-----------|------------|
| Звонки | VoIP через [pyVoIP](https://pypi.org/project/pyVoIP/) (SIP) или AT-модем SIM7600 |
| Распознавание речи | [Vosk](https://alphacephei.com/vosk/) (офлайн) |
| Синтез речи | [Piper TTS](https://github.com/rhasspy/piper) (офлайн, нейросеть) |
| SMS и интернет | SIM7600G-H по AT-командам (pyserial) |
| Контакты | SQLite + pymorphy3 (морфология, склонения, прозвища) |
| Нечёткий поиск | [rapidfuzz](https://github.com/rapidfuzz/RapidFuzz) + [navec](https://github.com/natasha/navec) |
| Команды | YAML-словарь + резолв имён через БД |

## Оборудование

- **Orange Pi Zero 2W** (4 GB RAM) или любой Linux ARM/x86
- **SIM7600G-H 4G HAT** -- подключение через USB (SMS + интернет)
- **USB-микрофон**
- **Динамик** (USB-аудио или 3.5 мм)

## Установка

### 1. Скачать и настроить (на компьютере, Windows/Linux/Mac)

```bash
git clone https://github.com/Volk-Arch/PaPaPhone.git
cd PaPaPhone
cp data/.env.example data/.env
# Отредактировать data/.env — SIP-логин, адрес, wake-фразы

# Windows:
pack.bat

# Linux/Mac:
bash pack.sh
```

### 2. Залить на устройство

```bash
scp papaphone-bundle.tar.gz user@device:~/
```

### 3. Установить на устройстве

```bash
tar xzf papaphone-bundle.tar.gz
cd papaphone-bundle
bash install.sh
sudo systemctl start papaphone
```

Всё. Сервис работает и запускается автоматически при включении.

## Запуск

```bash
# Продакшн
python -m src.main

# Отладка без модема и VoIP
python -m src.main --demo

# Без wake-фразы
python -m src.main --demo --no-wake
```

### Управление сервисом (после deploy.sh)

```bash
sudo systemctl start papaphone       # запустить
sudo systemctl stop papaphone        # остановить
sudo systemctl restart papaphone     # перезагрузить
sudo systemctl status papaphone      # статус
journalctl -u papaphone -f           # логи в реальном времени
journalctl -u papaphone --since today # логи за сегодня
```

Сервис запускается автоматически при включении устройства. При ошибке перезапускается через 10 секунд.

## Конфигурация

Файл `data/.env` (создаётся при `deploy.sh` или вручную из `data/.env.example`):

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `PAPAPHONE_CALL_MODE` | `voip` | Режим звонков: `voip` (SIP) или `modem` (AT) |
| `PAPAPHONE_SIP_SERVER` | | SIP-сервер (например `sip.zadarma.com`) |
| `PAPAPHONE_SIP_USER` | | SIP-логин |
| `PAPAPHONE_SIP_PASSWORD` | | SIP-пароль |
| `PAPAPHONE_SIP_PORT` | `5060` | SIP-порт |
| `PAPAPHONE_VOSK_MODEL` | `vosk-model-small-ru-0.22` | Папка модели Vosk |
| `PAPAPHONE_MODEM_PORT` | `/dev/ttyUSB0` | Порт SIM7600 (SMS + интернет) |
| `PAPAPHONE_WAKE_PHRASES` | `телефон,папафон` | Wake-фразы |
| `PAPAPHONE_HOME_ADDRESS` | *(не задан)* | Домашний адрес |
| `PAPAPHONE_NIGHT_START` | `23` | Начало ночного режима |
| `PAPAPHONE_NIGHT_END` | `9` | Конец ночного режима |
| `PAPAPHONE_AUDIO_INPUT_DEVICE` | *(системный)* | ALSA микрофон |
| `PAPAPHONE_AUDIO_OUTPUT_DEVICE` | *(системный)* | ALSA динамик |

Номер SOS (112) и экстренные контакты хранятся в БД. Контакт "Скорая" создаётся автоматически.
